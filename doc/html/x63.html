1.2. Installation

1.2.1. Obtaining the Latest Version

If you are reading this, you should have downloaded the module and already uncompressed it somewhere. If not, please goto dotProject's homepage at http://dotproject.net and follow the links to downloads and then to the dotmods download section at SourceForge.

1.2.2. Unpacking

You should have downloaded a file in the form of a compressed tarball. Copy the file to the modules directory of your dotProject installation and uncompress it there.

1.2.3. Verify and Adjust Ownership of Files

This manual assumes you are running on Linux, FreeBSD, MacOSX or other form of Unix or Unix-like operating system, with Apache 1, 2 or similar httpd server. If not, please refer to the appropriate documentation.

If you downloaded from the SourceForge site, all file permissions should be OK, but we advise you to double check for your particular http server configuration and security requirements. You must, however, make sure that the file ownership corresponds to the correct httpd username, group and other prerogatives. A simple chmod -R user.group unitcost/ will do from the modules directory.

1.2.4. Install the Module in dotProject

You must login to dotProject as the administrator and go to the Modules section of the administrative applications. Unitcost should be on the list as an un-installed and inactive module. You may now select the install link and then the activate link once it's installed. Do not make this module visible. This module is an add-on to the projects and tasks modules and has no screens of it's own, so there is no need, and it would probably break it, if you make it visible.

1.2.5. Installing the Patched Files

As mentioned earlier, there are some minor changes to dotProject's standard files that cannot be applied as a module. For this reason it is fully necessary to patch your version with the provided files. If you are in doubt of doing this, please refer to Section 1.4 for very precise details on the patched versions and why they are needed.

To install the patched files you must go into the unitcost module directory and locate the appropriate patch level. For example if you are running dotProject version 2.0.3 then the correct patch files will be located in the directory named patch_203. Inside this directory there will be subdirectories that correspond to your installed module directories.

Before copying the files to the actual module directories we recommend that you backup or rename these files in the original directories in case that you want to roll-back the installation. Take care when renaming files in each module's directories because the dotProject framework may interpret some file names automatically and try to load them. It is recommended you rename the files with a safe set of leading characters such as XXX_UNITCOST_foo.php where foo is the original filename, for example XXX_UNITCOST_addedit.php for the original addedit.php file.

Once you have backed up or renamed the original files, you may proceed to copy the patched versions into the corresponding directories.

1.2.6. Logout, Clear Browser Cache and Test

It is important that you logout from dotProject and clear all the cache or your browser to make sure that all the cached JavaScript is cleared away, before login in again. Up to Firefox version 1.0.7 you have to go to Edit->Preferences and then select Privacy and pressing Clear on Cache and Cookies. Newer versions of Firefox have the option directly on the Tools menu (I think).

Your new Unitcost module is ready to go. Just login and pull up a task to see the Unicost tab.
